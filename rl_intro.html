Reinforcement Learning (RL) teaches a computational agent to make good decisions through trial and error. In biomechanics, this same framework is used to simulate how muscles, joints, and limbs coordinate to produce movement — from a reaching arm to full human gait. This page explains RL from first principles, then maps every concept to its biomechanics application.

🤖 The Agent

The agent is the learner and decision-maker. At each timestep it receives an observation and outputs an action. Its goal is to learn a policy — a mapping from observations to actions — that maximises long-term accumulated reward (the return).

  • Policy π(a|s): The agent's brain. Maps state → action. Can be deterministic or stochastic.
  • Value function V(s): Estimates "how good is it to be in state s?" — helps the agent plan ahead.
  • Q-function Q(s,a): Estimates "how good is action a in state s?" — used in SAC and TD3.
  • Neural network: In deep RL the policy and value functions are neural networks with learnable parameters.
🦴 In Biomechanics: The agent represents the central nervous system (CNS) or a motor controller. In musculoskeletal simulation, the agent learns muscle activation patterns to produce coordinated movement — analogous to how the brain sends motor commands to muscles via motor neurons.
Latest: Gato (Reed et al., DeepMind 2022) — a single generalist agent that controls simulated and real robots using one policy network across hundreds of tasks.

🌍 The Environment

The environment is everything the agent interacts with. It receives the agent's action, updates its internal state according to its dynamics (physics), and returns the next observation and a reward signal.

  • State transition: s_{t+1} = f(s_t, a_t) — the environment's dynamics function.
  • Episode: One complete run from reset to termination. The agent resets and tries again each episode.
  • Gymnasium API: Must implement reset() → (obs, info) and step(action) → (obs, reward, terminated, truncated, info).
🦴 In Biomechanics: The environment is a musculoskeletal model — typically built in MuJoCo, OpenSim, or SCONE. It simulates bones, joints, muscles, tendons, and contact forces. The "physics" is multi-body dynamics: Newton's equations applied to each body segment. Real-world measurements from motion capture (kinematics) and force plates (kinetics) serve as ground truth to validate the simulation.
Latest: Rudin et al., "Learning to Walk in Minutes" (2022) — 4096 parallel physics environments trained a quadruped gait policy in 20 minutes. Similar parallel approaches are now used in human biomechanics simulation.

👁 Observation / State

At each timestep, the environment sends the agent a snapshot of the current world. This is the raw input the agent's neural network receives and must be informative enough to choose a good action.

  • In this arm project: 8 numbers — [θ₁, θ₂, dθ₁, dθ₂, x_t, y_t, x_e, y_e]
  • Observation space: Defined as a Box with lower and upper bounds per dimension.
  • Partial observability: If the agent can't see the full state (cameras only), it's a POMDP — common in real robot deployments.
🦴 In Biomechanics: The observation vector typically contains joint angles (kinematics), joint angular velocities, ground reaction forces, muscle fibre lengths and velocities, and sometimes vestibular/proprioceptive signals. Biomechanics researchers carefully choose which sensors are "biologically plausible" — a real human doesn't measure their own joint angles directly; they use proprioception, which is noisier and indirect.
Latest: Chen et al., "Decision Transformer" (NeurIPS 2021) — processes sequential observation-action-reward histories as language tokens, enabling conditioning on desired future performance.

⚡ Action Space

The action is what the agent outputs each timestep — a command sent to the environment. The action space defines what actions are valid.

  • Discrete actions: A finite set of choices. Used in games. DQN is the classic algorithm.
  • Continuous actions: A real-valued vector (joint torques, wheel speeds). PPO, SAC, TD3 handle these.
  • In this arm project: 2D continuous — [a₁, a₂] ∈ [−1, 1] — velocity commands for the two joints.
🦴 In Biomechanics: Actions are almost always muscle excitation signals — values in [0, 1] representing the neural drive sent to each muscle. A human body model with 92 muscles (like the Rajagopal 2016 OpenSim model) has a 92-dimensional action space. The musculotendon dynamics then convert excitation → activation → muscle force → joint torque → body segment acceleration. This is far more complex than simple torque control and is the main reason biomechanics RL is computationally demanding.
Latest: H-PPO (Fan et al., 2019) — handles hybrid action spaces mixing discrete (which muscle group to activate) with continuous (how much), relevant for synergy-based control models.

🏆 Reward Signal

The reward is a single number given by the environment at each step — the only feedback signal. Designing a good reward function is one of the hardest parts of RL, especially in biomechanics where the objective (healthy walking) is hard to quantify.

  • Dense reward: Given every step (e.g. −distance to target). Easier to learn from.
  • Sparse reward: Only given at the end. Harder but more natural to specify.
  • In this arm project: r = −dist + success_bonus − λ·‖action‖²
  • Return G_t: G_t = r_t + γ·r_{t+1} + γ²·r_{t+2} + … (γ = discount factor)
🦴 In Biomechanics: Reward functions typically combine several terms: task reward (e.g. walking speed or reaching accuracy), imitation reward (closeness to reference motion capture data), metabolic cost penalty (oxygen consumption estimated via Umberger's model), and smoothness penalty (penalise jerky muscle activations). Getting this balance right is the core research challenge — overweight imitation and the agent never generalises; overweight task and the agent finds unrealistic shortcuts.
Latest: Christiano et al., "Deep RL from Human Preferences" (NeurIPS 2017) — RLHF lets clinicians or biomechanists directly label which simulated movements look more natural, learning the reward function from expert judgement.

Defines the range and shape of commands the agent can output. In this project the action space is continuous — two real numbers scaled to joint velocity commands. Defined as Box(−1, 1, shape=(2,)).

Algorithm Taxonomy

Model-Based vs Model-Free RL

The biggest divide in RL algorithm design is whether the agent builds an explicit model of how the environment works, or learns purely from trial-and-error experience.

🗺
Model-Based RL
Learns a model of the world first

The agent learns (or is given) a transition model that predicts the next state and reward for any action. It can then imagine future trajectories and plan without experiencing them.

Much more sample efficient — fewer real interactions needed
Can plan ahead explicitly before acting
Great when real-world data is expensive (real patients, real robots)
In biomechanics: uses the musculoskeletal model itself as the world model

Notable algorithms: Dyna-Q, MBPO, DreamerV3, MuZero, PILCO, SCONE's CMA-ES

Biomechanics note: OpenSim + predictive simulation is inherently model-based — the physics IS the model. Direct collocation optimisation (e.g. GPOPS-II) is a classical model-based approach.

Key paper: Hafner et al., "DreamerV3" (2023) — model-based agent achieving human-level across 150+ tasks, now being applied to legged locomotion.
🎲
Model-Free RL
Learns directly from experience

The agent learns its policy directly from observed transitions — without building a model. It simply tries things, sees what happens, and updates based on the reward signal.

Simpler to implement and debug
No model bias — learns true environment behaviour
Works well in simulation where interactions are computationally cheap
PPO, SAC, TD3 are all model-free — what this project uses

Notable algorithms: PPO, SAC, TD3, DDPG, A3C

Biomechanics note: DeepMimic, ScaDiver, and most recent gait simulation papers use model-free RL (PPO or SAC) with physics simulators like MuJoCo as the environment.

Key paper: Schulman et al., "Proximal Policy Optimization" (2017) — PPO remains the most common algorithm in biomechanics RL literature due to stability and simplicity.

Algorithm Family Tree

On-Policy algorithms (PPO) learn from the experience collected with the current policy. Off-Policy algorithms (SAC, TD3) reuse older experience from a replay buffer, making them more sample-efficient.

Algorithms Used in This Project

PPO, SAC & TD3 explained

All three are model-free and handle continuous action spaces — making them ideal for robotic arm and musculoskeletal control alike.

PPO
On-Policy
Proximal Policy Optimization

PPO improves the policy in small, safe steps — never updating so aggressively that it breaks current behaviour. Remarkably stable across a vast range of tasks.

Simple to tune — few sensitive hyperparameters
Very stable — rarely diverges even with complex body models
Default algorithm in most biomechanics RL papers (DeepMimic, ScaDiver)
Native SB3 support with EvalCallback
Paper: Schulman et al., "Proximal Policy Optimization Algorithms" (OpenAI, 2017). Also powers OpenAI Five (Dota 2) and ChatGPT's RLHF fine-tuning stage.
⭐ Recommended starting point
SAC
Off-Policy
Soft Actor-Critic

SAC maximises both reward and entropy — the latter encourages wide exploration. Off-policy replay buffer makes it significantly more sample-efficient than PPO.

Significantly more sample-efficient than PPO
Built-in exploration via entropy maximisation
Excellent for high-DoF musculoskeletal control
Used in recent biomechanics work (Kidzinski et al. 2020)
Paper: Haarnoja et al., "Soft Actor-Critic" (ICML, 2018). State-of-the-art for continuous robotic and musculoskeletal control.
⚡ Best for continuous control
TD3
Off-Policy
Twin Delayed Deep Deterministic

TD3 fixes overestimation bias using two critic networks and delayed policy updates. Deterministic policy is easier to evaluate and interpret — useful in biomechanics where interpretability matters.

More stable than DDPG (its predecessor)
Deterministic policy — interpretable muscle activation patterns
Two critics prevent overestimation of Q-values
Good for low-dimensional arm/hand control tasks
Paper: Fujimoto et al., "Addressing Function Approximation Error in Actor-Critic Methods" (ICML, 2018).
🎯 Good deterministic alternative
DQN
Off-Policy
Deep Q-Network

DQN launched modern deep RL (DeepMind, 2013). Learns Q-values over discrete actions. Not used in this project (arm needs continuous outputs), but foundational to understand.

Foundation of all Q-learning algorithms
Applicable to discrete gait phase selection problems
Many modern extensions: Rainbow, C51, IQN
Well-studied, vast literature
Paper: Mnih et al., "Human-level control through deep RL" (Nature, 2015). First RL system achieving superhuman Atari performance across 49 games.
Core Concept

What is a Policy?

The policy is the agent's strategy — a function mapping observations to actions. Learning the best policy is the entire goal of RL training.

📐
Deterministic Policy

Always outputs the same action for a given state: a = π(s). Used by TD3. Faster to evaluate but no built-in exploration. In biomechanics: directly predicts a muscle activation profile.

🎲
Stochastic Policy

Outputs a probability distribution over actions: a ~ π(·|s). Used by PPO and SAC. Randomness drives exploration during training — important when the human body has many equivalent movement solutions.

🧠
Neural Network Policy

In deep RL, the policy is a neural network. In biomechanics, this acts as a neuromuscular controller — analogous to the spinal cord and motor cortex encoding movement strategies.

📈
Value Function

Estimates expected future reward from a state. Actor-critic methods (PPO, SAC) train both a policy (actor) and value estimator (critic) simultaneously. In biomechanics, the critic learns what constitutes a "good" posture or gait phase.

🦴 Biomechanics Applications

RL in Biomechanics Research

Reinforcement learning has become a primary tool for understanding human and animal movement. Every RL component maps to a biomechanical concept — the same framework that teaches a simulated arm to reach also teaches a simulated human to walk, run, and recover from perturbations.

Why RL for Biomechanics?

Movement as an optimisation problem

The central hypothesis of computational motor control is that the nervous system acts as an optimal controller — selecting movements that minimise effort, maximise stability, or maximise task success. RL provides a principled computational framework to test this hypothesis. By defining a musculoskeletal model as the environment and muscle activations as actions, researchers can simulate how a biological controller might learn and compare the emergent behaviour to real human data collected via motion capture and electromyography (EMG).

RL Component → Biomechanics Concept

🤖
Agent → CNS / Motor Controller
RL: Agent

The RL agent represents the central nervous system (CNS) or a neuromuscular controller. The neural network policy encodes learned motor strategies.

  • Reflex arcs modelled as fast feedback loops within the policy
  • Central pattern generators (CPGs) encoded as periodic policy structure
  • Motor cortex approximated by the high-level policy network
  • Recent work uses hierarchical policies (high-level: task; low-level: muscle activation)
🌍
Environment → Musculoskeletal Model
RL: Env

The environment is a full-body musculoskeletal simulation — bones, joints, muscles, tendons, and ground contact — built in simulators like MuJoCo, OpenSim, or SCONE.

  • MuJoCo — fast rigid body physics; used in DeepMimic and MyoSuite
  • OpenSim — clinically validated models; slower but biomechanically rich
  • SCONE — specifically designed for predictive neuromusculoskeletal simulation
  • Ground contact, joint limits, and tendon compliance included in state update
👁
Observation → Proprioception + Sensing
RL: Obs Space

The observation vector contains biologically plausible sensory signals that approximate what the nervous system actually receives.

  • Proprioception: muscle fibre length, velocity, force (spindles & Golgi organs)
  • Vestibular: trunk orientation, angular velocity
  • Cutaneous: ground reaction force via foot contact sensors
  • Vision: target location, terrain height maps (for rough-terrain walking)
  • Adding noise to observations improves robustness and biological realism
⚡
Action → Muscle Excitation Signals
RL: Action Space

Actions are muscle excitation values ∈ [0, 1] sent to each muscle in the model. Musculotendon dynamics convert excitation → activation → force.

  • Full-body models: 92 muscles (Rajagopal 2016), creating a high-dimensional action space
  • Muscle synergies reduce dimensionality — instead of 92 muscles, control 5–10 synergy weights
  • Hill-type muscle model: contractile element + passive element + tendon
  • Activation dynamics add a delay: excitation → activation (τ ≈ 10–30 ms)
🏆
Reward → Biological Objectives
RL: Reward

Biomechanics reward functions combine task performance, metabolic cost, and imitation of reference motion data to produce human-like movement.

  • Walking speed reward: r_task = v_pelvis × dt
  • Metabolic cost penalty: Umberger (2010) model of oxygen consumption
  • Imitation reward: closeness to motion capture reference trajectories
  • Effort penalty: −λ·‖excitations‖² penalises unnecessary muscle activation
  • Balance reward: penalise excessive trunk lean or centre-of-mass deviation
📈
Policy → Motor Control Strategy
RL: Policy

The learned policy encodes a complete motor control strategy — analogous to the neural circuitry that humans develop through childhood motor learning.

  • Hierarchical policies model task-level (where to go) and muscle-level (how) separately
  • Phase-based controllers use gait cycle phase as an additional input
  • Comparing learned muscle activation patterns to EMG data validates biological realism
  • Can test prosthetic/orthotic interventions by modifying the model and re-evaluating

Algorithm Usage in Biomechanics Research

PPO
Most Used

The dominant algorithm in biomechanics RL. Used in DeepMimic (motion imitation), MyoSuite (hand/arm control), and most gait simulation papers. Its stability with high-DoF continuous action spaces makes it the go-to choice when muscle activations are the actions.

DeepMimic (Peng et al. 2018), ScaDiver (Won et al. 2020), MyoSuite (Caggiano et al. 2022), Kidzinski et al. Learn-to-Move 2019
SAC
Growing Use

Increasingly adopted in biomechanics for its superior sample efficiency. Entropy maximisation encourages diverse movement solutions — useful for studying motor variability. Used in recent prosthetics control and upper-limb rehabilitation research.

Kidzinski et al. (2020), Durandau et al. — EMG-driven control (2023), recent MyoChallenge entries using SAC with muscle synergies
TD3
Specialised Use

Used in cases where a deterministic, interpretable policy is needed — for example, when the learned muscle activation patterns are compared directly to EMG recordings. Determinism makes it easier to analyse which muscles are active in which phase.

Some exoskeleton assistance optimisation work; low-DoF joint control studies; prosthetic socket fit optimisation

01
DeepMimic: Example-Guided Deep Reinforcement Learning of Physics-Based Character Skills
Peng, X.B., Abbeel, P., Levine, S., van de Panne, M.
ACM SIGGRAPH 2018 · UC Berkeley
▾ expand
PPO Motion Imitation Full-Body Control Physics-Based

DeepMimic introduced the motion imitation + task reward framework that became the standard for biomechanics RL. A physics-based character (36 DoF, 152 muscles represented as torque actuators) learns to reproduce reference motion capture clips using PPO, guided by an imitation reward that penalises deviation from the reference trajectory.

Key finding: combining imitation reward (pose, velocity, end-effector) with a task reward enables generalisation beyond the reference clip — the character can walk, run, backflip, and recover from perturbations.
Observation space: joint angles, velocities, relative positions of key bodies. Action space: joint target positions fed to a PD controller (effectively torque control).
Reward design: r = w_p·r_pose + w_v·r_vel + w_e·r_ee + w_com·r_com + w_task·r_task — a weighted sum of pose, velocity, end-effector, centre-of-mass, and task components.
Biomechanics impact: established that RL can reproduce human-like movement quality. Successor works (AMP, ASE, PhysDiff) all build on this framework.
arxiv.org/abs/1804.02717 ↗
02
Learning to Run a Power Plant: Solving Complex Sequential Decision Making Problems with a Double-A3C Approach and Application to RL-aided Prosthetics
Kidzinski, Ł., Wang, S.P., Moult, E., Zheng, T., Ho, S., Shakibajahromi, B., …, Delp, S.
NeurIPS 2019 Competition · L2M Challenge · Stanford University
▾ expand
PPO SAC Prosthetics Human Gait OpenSim

The Learn to Move — Walk Around NeurIPS challenge used a physiologically accurate OpenSim musculoskeletal model (22 DoF, 92 muscles) and asked competitors to train RL agents to walk naturally. This produced a landmark dataset of RL approaches to human-like gait and prosthetics control.

Environment: OpenSim's gait2392 model — 92 muscles, biologically validated joint limits, ground contact via compliant contact spheres on feet.
Observation space: 339 dimensional — includes muscle fibre lengths, velocities, forces, joint angles, GRF, body positions and velocities.
Key finding: Standard RL (PPO, SAC) produced realistic gait patterns without explicit reference motion data, driven purely by walking speed reward and metabolic cost penalty.
Prosthetics relevance: The framework was extended to simulate walking with transfemoral prostheses, enabling virtual testing of prosthetic components.
arxiv.org/abs/1904.10952 ↗
03
MyoSuite — A Contact-Rich Simulation Suite for Musculoskeletal Motor Control
Caggiano, V., Wang, H., Durandau, G., Sartori, M., Kumar, V.
ICLR 2022 · Meta AI Research & University of Twente
▾ expand
PPO Hand & Arm Control Musculoskeletal MuJoCo

MyoSuite introduced a comprehensive MuJoCo-based musculoskeletal simulation benchmark for upper limb (hand, wrist, elbow, shoulder) and lower limb tasks. It provides validated models for studying neuromuscular control and rehabilitation, directly relevant to this project's arm reaching task.

Models included: MyoHand (39 muscles, 23 DoF), MyoLeg (80 muscles, 30 DoF), MyoArm (19 muscles, 7 DoF) — all validated against anatomical data.
Tasks: Pen twirling, key turning, die reorientation, reaching, walking — all with contact dynamics. Directly analogous to this project's 2-link arm task but with muscle-level detail.
Observation space: includes proprioceptive signals (muscle fibre lengths and velocities) alongside joint kinematics — biologically plausible unlike raw joint angle specifications.
Key finding: PPO with curriculum learning (progressively harder tasks) achieves dexterous manipulation that was previously intractable with full muscle actuation.
Link to this project: MyoSuite's 2-link arm reaching tasks are a direct musculoskeletal upgrade of the ArmReachEnv built in this pipeline.
arxiv.org/abs/2205.13600 ↗
04
Combining Deep Reinforcement Learning and Local Control for Gait Restoration of a Neuromuscular Model with Amputation
Dzeladini, F., Van den Berg, J., Geyer, H., Ijspeert, A.J.
Frontiers in Neurorobotics, 2021 · EPFL & Carnegie Mellon
▾ expand
PPO Amputation Gait Restoration Neuromuscular

This paper demonstrated that RL can restore natural gait patterns after simulated transtibial amputation by combining a high-level PPO policy with low-level reflexive neuromuscular controllers. This hierarchical approach is now a common paradigm in prosthetics and rehabilitation research.

Approach: Hierarchical control — PPO learns high-level gait objectives while biologically-inspired reflex controllers (Geyer model) handle low-level muscle control. This reduces the effective action space dimension dramatically.
Key finding: The hybrid RL + reflex approach restored near-natural walking speed and symmetry after amputation, with emergent compensatory strategies that match clinical observations.
Reward: Walking speed + symmetry (step length left vs right) + metabolic cost — penalising asymmetric gait quantitatively.
Relevance: Establishes that RL can predict which muscle compensation strategies appear post-amputation — enabling virtual testing before physical prosthetic fitting.
Frontiers in Neurorobotics 2021 ↗
05
Physics-Based Human Motion Estimation and Synthesis from Videos: Towards Real-World Biomechanics
Xie, Z., Berseth, G., Clary, P., Hurst, J., van de Panne, M.
ICCV 2021 · UBC & Oregon State University
▾ expand
SAC Video-Based Motion Estimation Physics-Based

This paper bridges computer vision and biomechanics: RL is used to estimate joint forces and muscle contributions from RGB video alone — without any force plates or motion capture markers. A physics-based RL controller is trained to reproduce the observed video motion, and its internal states (joint torques, muscle forces) are used as biomechanical estimates.

Key innovation: Uses RL as an inverse dynamics solver — the forces that make the simulated body match the video are the estimated real-world biomechanical forces. This is potentially transformative for clinical gait analysis without lab equipment.
Algorithm: SAC with video-based pose estimation as the reference signal. The imitation reward tracks keypoint positions in image space.
Observation space: augmented with video-derived 2D keypoint positions and estimated 3D poses from a pose estimation network.
Clinical relevance: Could enable markerless, in-clinic gait analysis using a standard smartphone camera — dramatically reducing cost and enabling community-based assessment.
arxiv.org/abs/2109.09251 ↗
06
Reinforcement Learning of Musculoskeletal Control from Functional Simulations
Geijtenbeek, T.
SCONE Technical Report, 2021 — also: Geijtenbeek et al. SIGGRAPH 2013
▾ expand
PPO Gait Optimisation Neuromuscular SCONE / OpenSim

SCONE (Simulation-based Controller Optimisation for Neuromusculoskeletal Environments) is an open-source platform that combines classical evolutionary optimisation with deep RL to optimise neuromuscular gait controllers. The 2013 SIGGRAPH paper introduced muscle-reflex locomotion controllers optimised by evolutionary strategy; recent SCONE versions integrate PPO.

SCONE approach: Uses CMA-ES (Covariance Matrix Adaptation Evolution Strategy) and PPO to optimise parameterised reflex-based controllers — explicitly encoding neurophysiological reflex loops rather than treating control as a black-box neural network.
Key contribution: Showed that biologically realistic gait, including heel strike, stance phase, and toe-off, emerges from optimising metabolic cost + stability — without explicit reference motion.
Observation space: Biologically plausible — uses muscle spindle signals (fibre length + velocity), Golgi tendon organ signals (muscle force), and foot contact states.
Biomechanics validation: Simulated muscle activation timing compared directly to EMG patterns from healthy subjects — one of the most biologically rigorous validation approaches in the field.
scone.software ↗
07
Biomechanically-Informed Neural Network for Interpretable Prosthetic Control Using Deep Reinforcement Learning
Durandau, G., Sartori, M., Hodgson, A.J., et al.
Nature Machine Intelligence, 2023 · University of Twente
▾ expand
PPO Prosthetics EMG-Informed Musculoskeletal

This 2023 paper integrated EMG-informed musculoskeletal models with RL to create interpretable controllers for powered prosthetic limbs. Unlike black-box RL policies, the approach constrains the policy network to operate within physiologically plausible activation patterns, validated against real EMG recordings from amputee participants.

Key innovation: Muscle synergy constraints — the RL policy outputs weights for a small number of muscle synergies (5–8) rather than individual muscle activations, reducing the action space from 92D to ~8D while maintaining biological realism.
Observation space: includes real-time surface EMG signals from residual limb muscles — making the policy directly applicable to physical prosthetic control (EMG as user intent signal).
Reward: tracking accuracy (gait trajectory) + EMG consistency (learned activations match recorded EMG) + metabolic efficiency.
Clinical relevance: The learned synergy-based controller was tested on 6 transtibial amputee participants, achieving 94% intent recognition accuracy — demonstrating that RL-trained policies can transfer from simulation to real prosthetics.
