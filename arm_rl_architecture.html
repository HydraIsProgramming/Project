1) Arm Model Architecture

Planar serial manipulator with two revolute joints and fixed rigid links.

Base and Joint 1 (Shoulder)
  • Base at origin: (0, 0)
  • Joint angle: theta1
  • Controls link-1 orientation
Joint 2 (Elbow)
  • Joint angle: theta2 (relative)
  • Controls link-2 orientation
  • Absolute link-2 angle: theta1 + theta2
End Effector and Target
  • End-effector position: (x_e, y_e)
  • Target position: (x_t, y_t)
  • Task: reach and hold target at horizontal pose

2) Constraints

Physical Constraints
  • Rigid links with fixed lengths: L1, L2 (no extension/compression).
  • Joint-angle bounds: theta1, theta2 in [-pi, pi] after wrapping.
  • Joint velocity bounds: dtheta1, dtheta2 in [-max_vel, +max_vel].
  • Reachable annulus for valid target sampling: r in [abs(L1-L2)+margin, L1+L2-margin].
RL/Task Constraints
  • Action bounds: a = [a1, a2], each in [-1, 1].
  • Time discretization: fixed dt per step.
  • Episode ends on success hold or max_steps.
  • Horizontal hold condition uses absolute link-2 orientation near 0 rad.

3) Mathematical Formulation

Forward Kinematics
elbow_x = L1 * cos(theta1) elbow_y = L1 * sin(theta1) x_e = L1 * cos(theta1) + L2 * cos(theta1 + theta2) y_e = L1 * sin(theta1) + L2 * sin(theta1 + theta2)
Distance and Gradient
d_t = sqrt((x_t - x_e)^2 + (y_t - y_e)^2) gradient_t = (d_t - d_{t-1}) / dt horizontal_error = abs(wrap_to_pi(theta1 + theta2))
Discrete Dynamics (velocity control)
dtheta = clip(dtheta + action * max_vel, -max_vel, max_vel) theta = wrap_to_pi(theta + dtheta * dt)

5) Action Space

Continuous 2D command
action = [a1, a2], where each a_i in [-1, 1] u = action * max_vel a1 -> shoulder velocity command a2 -> elbow velocity command

Policy output can be deterministic or stochastic; during simulation, deterministic policy is preferred for stable visualization.

6) Reward Function (Reach + Hold + Horizontal + Smooth Gradient)

r_t = - w_d * d_t - w_g * abs(gradient_t) - w_a * ||action_t||^2 - w_hz * horizontal_error + w_s * I(success_radius) + w_hold * I(success_radius AND horizontal_error < eps_hz for N_hold steps)
  • - w_d * d_t: drives end-effector toward target.
  • - w_g * abs(gradient_t): encourages low gradient magnitude (smooth approach, less oscillation).
  • - w_a * ||action||^2: regularizes control effort.
  • - w_hz * horizontal_error: enforces horizontal orientation at target.
  • + w_hold term: rewards stable hold at desired pose, not only first contact.

7) Agent-Environment Interaction Flow (Detailed Contract)

Agent Subsystem
Environment Subsystem

0) Reset / State

Episode init and memory reset.

Details: Initialize theta, dtheta, sample reachable target, and reset memory variables: d_(t-1), hold counter, step counter.

1) Observation Space O_t

9D input vector to policy.

Details: [theta1, theta2, dtheta1, dtheta2, x_t, y_t, x_e, y_e, gradient_t].
gradient_t = (d_t - d_(t-1))/dt.
Angles are wrapped and velocities clipped.

2) Policy / Agent

Maps observation to action.

Details: The actor computes a_t ~ pi(a|obs_t) (or deterministic in simulation), optimized to maximize expected discounted return.

3) Action Space A_t

2D bounded command.

Details: action_t = [a1, a2], each in [-1, 1]. These are shoulder/elbow velocity control signals before scaling.

4) Command + Safety

Scale and enforce limits.

Details: Scale by max_vel, clip joint velocities, and wrap angles to [-pi, pi]. Rigid-link constraint preserved.

5) Dynamics Update

Discrete-time state transition.

Details: Euler update: dtheta = clip(dtheta + u_t), theta = wrap(theta + dtheta*dt).

6) Kinematics + Errors

Compute end-effector and errors.

Details: Forward kinematics gives (x_e, y_e), then compute d_t, gradient_t, and horizontal_error = |wrap(theta1+theta2)|.

7) Cost Function C_t

Explicit objective to minimize.

Details: C_t = w_d*d_t + w_g*|gradient_t| + w_a*||a_t||^2 + w_hz*horizontal_error. Optional terms may penalize overshoot or unsafe motion.

8) Reward Function R_t

RL optimization signal.

Details: R_t = -C_t + success_bonus + hold_bonus. Reward encourages reach, horizontal posture, low gradient, and stable hold.

9) Step Output

obs, reward, done flags.

Details: Emit Gym tuple: (obs_(t+1), reward_t, terminated, truncated, info). If not done, loop back with obs_(t+1).

Agent I/O

  • Input: observation_t (9D vector including gradient_t)
  • Output: action_t in R^2
  • Internal: policy network parameters and optimizer state

Environment I/O

  • Input: action_t
  • Output: observation_{t+1}, reward_t, terminated, truncated, info
  • Internal: arm state, target state, previous distance for gradient

Observation Component Detail

  • Joint state: theta1, theta2, dtheta1, dtheta2
  • Task state: x_t, y_t
  • Kinematic output: x_e, y_e
  • Progress signal: gradient_t

Reward Component Detail

  • Distance minimization and hold completion
  • Horizontal posture regularization
  • Control effort regularization
  • Gradient smoothness regularization

Cost Function Detail

  • Cost is explicit and reward is derived as negative cost plus bonuses.
  • Separating cost improves interpretability and tuning of each penalty term.
  • Cost terms: distance, gradient magnitude, control effort, horizontal error, optional overshoot.
  • Lower cost trajectory naturally yields steeper, stable learning improvement.
